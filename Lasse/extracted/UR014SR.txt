User Requirement 014:
	STATEMENT The system's interface shall be windows-based
		PRIORITY A

System Requirement 014.1:
	STATEMENT A conference shall look like a window frame
			  with a bar above that contains the names of the used plugins.
			  The bar shall have the same length as the window frame.
		PRIORITY A

System Requirement 014.2:
	STATEMENT The position of the window frame shall not be absolute, the user shall be able move the window inside of the browser
		PRIORITY A

System Requirement 014.3:
	STATEMENT When the window frame is moved, the new position shall be kept as actual position of the window
		PRIORITY A

System Requirement 014.4:
	STATEMENT With several window frames opened, the focused one shall be always in front of all other window frames
		PRIORITY A

System Requirement 014.5:
	STATEMENT The inactive window frames shall cover up in a row, where the last focused covers the pre-last-focused(and so on)
		PRIORITY A

System Requirement 014.6:
	STATEMENT The plugin shall be executed inside its window frame
		PRIORITY A

System Requirement 014.7:
	STATEMENT There shall be buttons in the frame-top-bar to give the ability to minimize,close and maximize the window frame.
		PRIORITY A

System Requirement 014.8:
	STATEMENT The moderator shall be able to use a configuration button inside of the conference window.
			  This button shall open an conference menu bar with at least following functions:
									- invite User(A)
									- remove User(A)
									- give rights(B)
									- remove rights(B)
		PRIORITY A/B
